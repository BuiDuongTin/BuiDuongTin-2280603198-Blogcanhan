Tối ưu hóa hiệu suất JavaScript trong ứng dụng web

20/08/2025 19 phút đọc JavaScript

Tầm quan trọng của hiệu suất trong ứng dụng web

Hiệu suất là một trong những yếu tố quan trọng nhất ảnh hưởng đến trải nghiệm người dùng trên web. Theo nghiên cứu, 53% người dùng sẽ rời khỏi trang web nếu nó mất hơn 3 giây để tải, và 79% người dùng sẽ không quay lại trang web có hiệu suất kém. Hơn nữa, Google đã đưa tốc độ tải trang thành một trong những yếu tố xếp hạng SEO.

JavaScript là một trong những yếu tố chính ảnh hưởng đến hiệu suất của ứng dụng web. Trong bài viết này, chúng ta sẽ khám phá các kỹ thuật tối ưu hóa JavaScript từ cơ bản đến nâng cao, giúp ứng dụng web của bạn chạy nhanh hơn và tiêu tốn ít tài nguyên hơn.

Hiểu về cách trình duyệt xử lý JavaScript

Để tối ưu hiệu suất JavaScript, điều quan trọng là hiểu cách trình duyệt tải và thực thi mã JavaScript. Quá trình này thường bao gồm các bước sau:

  1. Tải (Downloading): Trình duyệt tải file JavaScript từ server
  2. Phân tích (Parsing): Biến đổi mã nguồn thành AST (Abstract Syntax Tree)
  3. Biên dịch (Compilation): Biên dịch AST thành bytecode hoặc machine code
  4. Thực thi (Execution): Chạy mã đã được biên dịch

Mỗi bước này đều có thể ảnh hưởng đến hiệu suất trang web. Hãy xem xét các kỹ thuật để cải thiện hiệu suất ở từng bước.

Tối ưu thời gian tải

1. Giảm kích thước file

Kích thước file JavaScript càng nhỏ, thời gian tải càng nhanh.

Minification (Thu nhỏ mã)

Minification là quá trình loại bỏ các ký tự không cần thiết (khoảng trắng, xuống dòng, comment) và rút gọn tên biến để giảm kích thước file. 


// Trước khi minify
function calculateTotal(price, quantity, taxRate) {
    // Tính tổng giá trị
    var subtotal = price * quantity;
    var tax = subtotal * taxRate;
    var total = subtotal + tax;
    
    return total;
}

// Sau khi minify
function calculateTotal(a,b,c){var d=a*b,e=d*c;return d+e}

Công cụ phổ biến: UglifyJS, Terser, Closure Compiler

Compression (Nén)

Nén file bằng gzip hoặc brotli trước khi gửi từ server đến client giúp giảm kích thước tải xuống. 


// Cấu hình nén gzip trên Express.js
const express = require('express');
const compression = require('compression');
const app = express();

// Áp dụng middleware nén
app.use(compression());

// Serve static files
app.use(express.static('public'));

app.listen(3000);

2. Code splitting (Phân chia mã)

Code splitting giúp chia nhỏ mã JavaScript thành các gói (bundles) nhỏ hơn, chỉ tải những gì cần thiết cho trang hiện tại. 


// Sử dụng dynamic import trong JavaScript hiện đại
button.addEventListener('click', async () => {
    // Chỉ tải module khi cần thiết
    const { default: Chart } = await import('./chart.js');
    
    // Sử dụng module
    const chart = new Chart(data);
    chart.render();
});

// Với webpack
// Cấu hình trong webpack.config.js
module.exports = {
  entry: './src/index.js',
  output: {
    filename: 'main.[contenthash].js',
    chunkFilename: 'chunk.[contenthash].js',
    path: path.resolve(__dirname, 'dist')
  },
  optimization: {
    splitChunks: {
      chunks: 'all'
    }
  }
};

3. Tree shaking

Tree shaking là kỹ thuật loại bỏ mã không sử dụng (dead code) khỏi bundle cuối cùng. 


// Trước tree shaking
import { sum, multiply, divide } from './math.js';

// Chỉ sử dụng hàm sum
console.log(sum(5, 3));

// Sau tree shaking, chỉ hàm sum được đưa vào bundle, 
// multiply và divide bị loại bỏ

4. Lazy loading (Tải chậm)

Lazy loading là kỹ thuật tải tài nguyên khi cần thiết thay vì tải tất cả ngay từ đầu. 


// Lazy loading images
document.addEventListener("DOMContentLoaded", function() {
    const lazyImages = [].slice.call(document.querySelectorAll("img.lazy"));
    
    if ("IntersectionObserver" in window) {
        const lazyImageObserver = new IntersectionObserver(function(entries, observer) {
            entries.forEach(function(entry) {
                if (entry.isIntersecting) {
                    let lazyImage = entry.target;
                    lazyImage.src = lazyImage.dataset.src;
                    lazyImage.classList.remove("lazy");
                    lazyImageObserver.unobserve(lazyImage);
                }
            });
        });
        
        lazyImages.forEach(function(lazyImage) {
            lazyImageObserver.observe(lazyImage);
        });
    }
});

// HTML
<img class="lazy" data-src="image.jpg" src="placeholder.jpg" alt="Lazy loaded image">

1. Tối ưu DOM manipulation

Thao tác với DOM là một trong những nguyên nhân chính gây ra hiệu suất kém trong ứng dụng web.

Giảm thiểu reflow và repaint


// Không tối ưu - nhiều reflow
for (let i = 0; i < 1000; i++) {
    document.getElementById("container").innerHTML += `<div>Item ${i}</div>`;

// Tối ưu - một reflow duy nhất
const fragment = document.createDocumentFragment();

    fragment.appendChild(div);
}

2. Event delegation (Ủy thác sự kiện)

Event delegation giúp giảm số lượng event listener bằng cách gắn sự kiện vào phần tử cha thay vì nhiều phần tử con. 


// Không tối ưu - gắn event cho mỗi button
document.querySelectorAll('.btn').forEach(button => {
    button.addEventListener('click', function() {
        console.log('Button clicked:', this.textContent);
    });
});

// Tối ưu - sử dụng event delegation
document.getElementById('button-container').addEventListener('click', function(event) {
    if (event.target.matches('.btn')) {
        console.log('Button clicked:', event.target.textContent);
    }
});

3. Tránh memory leaks

Memory leaks (rò rỉ bộ nhớ) xảy ra khi bộ nhớ không được giải phóng đúng cách, dẫn đến hiệu suất giảm dần theo thời gian. 


// Rò rỉ bộ nhớ
function setupListener() {
    const element = document.getElementById('myButton');
    element.addEventListener('click', function() {
        // Đây là một closure, giữ tham chiếu đến biến largeData
        console.log(largeData);
    });
    
    // Biến này sẽ không được thu gom rác vì closure giữ tham chiếu
    const largeData = new Array(10000000).fill('data');
}

// Khắc phục
function setupListenerFixed() {
    const element = document.getElementById('myButton');
    
    function handleClick() {
        console.log('Button clicked');
    }
    
    element.addEventListener('click', handleClick);
    
    // Xóa event listener khi không cần nữa
    return function cleanup() {
        element.removeEventListener('click', handleClick);
    };
}

// Sử dụng
const cleanup = setupListenerFixed();
// Khi không cần nữa
cleanup();

4. Sử dụng Web Workers

Web Workers cho phép chạy JavaScript trong luồng riêng biệt, không chặn UI thread. 


// main.js
// Tạo worker
const worker = new Worker('worker.js');

// Gửi dữ liệu đến worker
worker.postMessage({
    type: 'PROCESS_DATA',
    data: largeArray
});

// Xử lý kết quả từ worker
worker.onmessage = function(event) {
    const result = event.data;
    console.log('Processed data:', result);
};

// worker.js
// Nhận dữ liệu từ main thread
self.onmessage = function(event) {
    const { type, data } = event.data;
    
    if (type === 'PROCESS_DATA') {
        // Xử lý dữ liệu tốn thời gian
        const result = processLargeData(data);
        
        // Gửi kết quả về main thread
        self.postMessage(result);
    }
};

function processLargeData(data) {
    // Xử lý phức tạp, tốn CPU
    return data.map(item => item * 2).filter(item => item > 100);
}

5. Memoization (Nhớ kết quả)

Memoization là kỹ thuật lưu trữ kết quả của các hàm tốn kém để tái sử dụng, thay vì tính toán lại. 


// Không sử dụng memoization - tính toán mỗi lần gọi
function fibonacci(n) {
    if (n <= 1) return n;
    return fibonacci(n - 1) + fibonacci(n - 2);
}

// Sử dụng memoization
function memoizedFibonacci() {
    const cache = {};
    
    return function fib(n) {
        if (n in cache) {
            return cache[n];
        }
        
        if (n <= 1) {
            return n;
        }
        
        const result = fib(n - 1) + fib(n - 2);
        cache[n] = result;
        return result;
    };
}

const fastFibonacci = memoizedFibonacci();
console.time('Không memoize');
fibonacci(40); // Rất chậm
console.timeEnd('Không memoize');

console.time('Có memoize');
fastFibonacci(40); // Nhanh hơn rất nhiều
console.timeEnd('Có memoize');

JavaScript chặn quá trình render của trình duyệt. Việc tải và thực thi JavaScript không phù hợp có thể làm chậm trang web.

<!-- Đặt script không quan trọng ở cuối -->

2. Tối ưu với requestAnimationFrame

requestAnimationFrame giúp đồng bộ hóa các animation và visual updates với chu kỳ render của trình duyệt. 


// Không tối ưu - có thể gây giật lag
function animate() {
    element.style.transform = `translateX(${position}px)`;
    position += 5;
    setTimeout(animate, 16); // ~60fps
}

// Tối ưu với requestAnimationFrame
function animateSmooth() {
    element.style.transform = `translateX(${position}px)`;
    position += 5;
    requestAnimationFrame(animateSmooth);
}

requestAnimationFrame(animateSmooth);

1. Browser rendering optimization

Hiểu và tối ưu quá trình render của trình duyệt có thể cải thiện đáng kể hiệu suất.

Tối ưu Critical Rendering Path

  .hero { /* styles */ }
</style>

<!-- Preload fonts -->

Tránh forced synchronous layouts


// Không tối ưu - forced synchronous layout
function resizeAllElements() {
    const elements = document.querySelectorAll('.box');
    
    elements.forEach(el => {
        // Đọc (forces layout computation)
        const height = el.offsetHeight;
        
        // Ghi (triggers layout again)
        el.style.height = (height * 2) + 'px';
    });
}

// Tối ưu - tách đọc và ghi DOM
function resizeAllElementsOptimized() {
    const elements = document.querySelectorAll('.box');
    const heights = [];
    
    // Đọc tất cả
    elements.forEach(el => {
        heights.push(el.offsetHeight);
    });
    
    // Ghi tất cả
    elements.forEach((el, i) => {
        el.style.height = (heights[i] * 2) + 'px';
    });
}

2. Tối ưu JavaScript Framework

Các framework như React, Vue, Angular có các kỹ thuật tối ưu hiệu suất riêng.

React


// Tối ưu với memo
const MemoizedComponent = React.memo(function MyComponent(props) {
    // Chỉ render lại khi props thay đổi
    return <div>{props.name}</div>;
});

// Tối ưu với useMemo và useCallback
function OptimizedComponent({ data, onItemClick }) {
    // Tránh tính toán lại khi re-render
    const processedData = React.useMemo(() => {
        return heavyProcessing(data);
    }, [data]);
    
    // Tránh tạo lại function khi re-render
    const handleClick = React.useCallback((item) => {
        onItemClick(item.id);
    }, [onItemClick]);
    
    return (
        <ul>
            {processedData.map(item => (
                <li key={item.id} onClick={() => handleClick(item)}>
                    {item.name}
                </li>
            ))}
        </ul>
    );
}

3. Service Workers và Caching

Service Workers cho phép cache tài nguyên và thậm chí cho phép ứng dụng hoạt động offline. 


// Đăng ký Service Worker
if ('serviceWorker' in navigator) {
    window.addEventListener('load', () => {
        navigator.serviceWorker.register('/sw.js')
            .then(registration => {
                console.log('ServiceWorker đăng ký thành công:', registration.scope);
            })
            .catch(error => {
                console.log('Đăng ký ServiceWorker thất bại:', error);
            });
    });
}

// sw.js - Service Worker
const CACHE_NAME = 'my-site-v1';
const urlsToCache = [
    '/',
    '/styles/main.css',
    '/scripts/main.js',
    '/images/logo.png'
];

// Cài đặt (install) và cache tài nguyên
self.addEventListener('install', event => {
    event.waitUntil(
        caches.open(CACHE_NAME)
            .then(cache => {
                console.log('Opened cache');
                return cache.addAll(urlsToCache);
            })
    );
});

// Xử lý fetch requests
self.addEventListener('fetch', event => {
    event.respondWith(
        caches.match(event.request)
            .then(response => {
                // Trả về từ cache nếu có
                if (response) {
                    return response;
                }
                
                // Ngược lại, fetch từ network
                return fetch(event.request)
                    .then(response => {
                        // Kiểm tra response hợp lệ
                        if (!response || response.status !== 200 || response.type !== 'basic') {
                            return response;
                        }
                        
                        // Clone response
                        const responseToCache = response.clone();
                        
                        // Cache cho lần sau
                        caches.open(CACHE_NAME)
                            .then(cache => {
                                cache.put(event.request, responseToCache);
                            });
                            
                        return response;
                    });
            })
    );
});

Checklist tối ưu hiệu suất JavaScript

Dưới đây là checklist tổng hợp các kỹ thuật tối ưu hiệu suất JavaScript:

Tối ưu delivery

  • ✅ Minify và nén JavaScript
  • ✅ Sử dụng HTTP/2
  • ✅ Implement code splitting
  • ✅ Sử dụng tree shaking
  • ✅ Áp dụng các resource hints (preload, prefetch)
  • ✅ Sử dụng CDN

Tối ưu thực thi

  • ✅ Giảm thiểu DOM operations
  • ✅ Sử dụng event delegation
  • ✅ Tránh forced reflow/repaint
  • ✅ Memoize kết quả của hàm phức tạp
  • ✅ Sử dụng Web Workers cho tác vụ nặng
  • ✅ Áp dụng virtual scrolling cho danh sách lớn

Tối ưu rendering

  • ✅ Tải JavaScript với async/defer
  • ✅ Ưu tiên critical CSS inline
  • ✅ Sử dụng requestAnimationFrame cho animations
  • ✅ Tách UI updates thành chunks nhỏ

Lập trình hiệu quả

  • ✅ Tránh memory leaks
  • ✅ Sử dụng cấu trúc dữ liệu phù hợp
  • ✅ Tránh micro-optimizations quá sớm
  • ✅ Đo đạc trước và sau khi tối ưu

Kết luận

Tối ưu hiệu suất JavaScript là một phần quan trọng của quá trình phát triển web hiện đại. Nó không chỉ cải thiện trải nghiệm người dùng mà còn giúp tăng tỷ lệ chuyển đổi, cải thiện SEO và giảm tải cho server.

Hãy nhớ rằng tối ưu hiệu suất là một quá trình liên tục, không phải một việc làm một lần. Các trình duyệt, thiết bị và kỳ vọng của người dùng liên tục thay đổi, vì vậy việc theo dõi và cải thiện hiệu suất nên là một phần của quy trình phát triển thường xuyên.

Bắt đầu bằng việc đo lường hiệu suất hiện tại, xác định các vấn đề, áp dụng các kỹ thuật tối ưu phù hợp, và đánh giá kết quả. Với cách tiếp cận có hệ thống, bạn có thể đạt được những cải thiện đáng kể về hiệu suất và mang lại trải nghiệm web tuyệt vời cho người dùng.
